💻 CU-01 — Generar Turno de Atención y Perfilamiento
ID CU-01
Nombre Generar Turno de Atención y Perfilamiento
Descripción Permite al ciudadano ingresar sus datos, ser perfilado según su grupo poblacional y seleccionar el servicio requerido para ingresar a la cola de espera.
Actor Usuario General (Ciudadano / NIT de Empresa)
Precondiciones El kiosco debe mostrar la pantalla de bienvenida y términos de uso de datos. Vigilante capacitado disponible para soporte opcional.
Flujo Normal 1. El usuario acepta términos y condiciones.
2. Selección de Perfil: General, Víctima, Prioritario o Empresario.
3. Selección de Tipo de Atención: "Normal" o "Especial".
4. Selección de Documento: CC, TI o CE.
5. Ingreso de número de identificación.
6. Validación de Registro: Si no tiene registro en la APE, el sistema permite continuar pero emite advertencia.
7. Ingreso de Teléfono (Opcional).
8. Selección de Servicio: Orientación Laboral, Formación Académica o Emprendimiento.
9. El sistema genera el turno con código alfanumérico según prioridad y lo guarda en la BD.
Postcondiciones Se registra el turno en estado "Espera" y se libera la interfaz del kiosco.
Flujos Alternativos 4.a: Si el número ya tiene un turno activo, el sistema notifica error: "Solo puedes solicitar un (1) turno por día".
4.b: Si la longitud del documento no cumple la normativa colombiana (CC/TI: 8-10 dígitos, CE: mínimo 6), solicita corregir.
5.a: El usuario deja el campo de teléfono vacío y presiona "Continuar" (flujo permitido).
Flujo Principal El sistema valida los datos de identificación, asigna un código alfanumérico según la prioridad (V/E/P/G) y lo ingresa a la base de datos de la APE con estado "Espera".

👤 CU-02 — Gestión de Atención al Ciudadano
ID CU-02
Nombre Gestión de Atención al Ciudadano
Descripción El asesor solicita un turno de la fila, lo atiende y actualiza la información en la pantalla pública.
Actor Usuario Asesor (OT / OV / AT)
Precondiciones El asesor debe estar logueado y en estado "Disponible" (sin atención activa y sin pausa activa).
Flujo Normal 1. El asesor presiona "Llamar Siguiente Turno".
1.1 El sistema busca el turno prioritario más antiguo según el tipo de asesor (OT: Prioritario→General 3:1 / OV: Empresario→Víctima / AT: todos).
1.2 El sistema muestra los datos del ciudadano en el panel del asesor.
1.3 El sistema actualiza la pantalla pública con audio (chime + voz).
2. El asesor realiza el trámite con el ciudadano.
3. El asesor presiona "Finalizar Atención".
4. El sistema registra hora de fin y cambia el estado del turno a "Finalizado".
Postcondiciones El turno cambia a estado "Finalizado", el asesor queda libre y se registran los tiempos (atnc_hora_inicio / atnc_hora_fin) para reportes de la coordinadora.
Flujos Alternativos 3.a: El ciudadano no se presenta → El asesor presiona "Ciudadano Ausente" → El sistema cambia el estado a "Ausente" y registra la hora de fin.
3.b: El asesor intenta llamar un nuevo turno con atención activa → El botón "Llamar Siguiente" no aparece hasta finalizar la atención actual.
Flujo Principal El sistema vincula el turno con el módulo del asesor mediante lockForUpdate(), actualiza los tiempos de respuesta en tiempo real y notifica a la sala de espera vía polling.

☕ CU-03 — Registrar Receso Temporal
ID CU-03
Nombre Registrar Receso Temporal (Pausa)
Descripción Permite al asesor ausentarse temporalmente sin que el sistema le asigne nuevos turnos, informando su estado a la coordinación.
Actor Orientador Técnico (OT) / Orientador de Víctimas (OV) / Asesor Total (AT)
Precondiciones El asesor no debe tener un turno activo en ese momento. Máximo 3 recesos por jornada.
Flujo Normal 1. El asesor selecciona "Iniciar Receso" en el sidebar.
2. El sistema verifica que no haya atención activa.
3. El sistema registra la hora de inicio en tabla pausas_asesor.
4. El estado del módulo cambia a "Pausa" — no recibe nuevos turnos.
5. Al regresar, el asesor presiona "Finalizar Receso".
6. El sistema calcula la duración (diffInMinutes) y guarda en BD.
7. El estado vuelve a "Activo" — el módulo vuelve a recibir turnos.
Postcondiciones Se guarda el registro de duración de la pausa en tabla pausas_asesor para el reporte de productividad de la coordinadora.
Flujos Alternativos 1.a: Si intenta pausar mientras atiende a alguien → Sistema muestra: "No puedes iniciar un receso mientras tienes una atención activa".
1.b: Si ya tiene 3 recesos en el día → Sistema muestra: "Has alcanzado el límite máximo de 3 recesos permitidos por día".
1.c: Si ya tiene un receso activo → Sistema muestra: "Ya tienes un receso activo en curso".
Flujo Principal El sistema bloquea la recepción de turnos en el módulo mientras el asesor está en receso, asegurando que la pantalla no llame a nadie a un módulo vacío.

🛡 CU-04 — Visualizar Reportes y Gestión de Flujo (Coordinación)
ID CU-04
Nombre Visualizar Reportes y Gestión de Flujo (Coordinación)
Descripción Herramienta para la coordinación donde se supervisa la operatividad de los módulos y el cumplimiento de metas de atención.
Actor Coordinador
Precondiciones El coordinador debe estar autenticado con credenciales de supervisión y tener acceso al panel administrativo.
Flujo Normal 1. El coordinador ingresa al Dashboard de Coordinación.
2. El sistema carga KPIs: turnos hoy, en espera, en atención, finalizados, ausentes, tiempo medio.
3. Supervisa que los módulos 15 y 19 cumplan la ruta de víctimas (Meta: ~6 emprendedores/semana).
4. Controla los tiempos de espera — Alerta si superan los 20 minutos.
5. Verifica la rotación bimestral del personal de apoyo (días restantes ≤ 60).
6. Analiza gráficas de flujo por hora (Chart.js).
7. Genera reporte diario exportable a Excel.
Postcondiciones Generación de métricas de productividad. Registro de incidencias por tiempos excedidos. Actualización del historial de rotación de personal. Consolidación de datos para informes gerenciales.
Flujos Alternativos 2.a: Si un turno supera los  15 minutos en espera → El sistema emite alerta visual crítica en el Dashboard (color rojo).
2.b: Si el conteo de emprendedores es inferior al proyectado → El indicador se marca en color de advertencia.
2.c: Si el personal de vigilancia supera el periodo bimestral → El sistema genera recordatorio de gestión administrativa.
Flujo Principal El coordinador ingresa al módulo de reportes. El sistema valida permisos y despliega información actualizada. Se verifica cumplimiento de metas en módulos 15 y 19. Se auditan tiempos de respuesta y pertinencia del servicio según perfil del usuario.

🔍 CU-05 — Consulta de Estado de Turno
ID CU-05
Nombre Consulta de Estado de Turno
Descripción Permite a un ciudadano verificar la información y el estado actual de su turno (Espera, Atendiendo, etc.) en tiempo real mediante su documento de identidad, sin necesidad de iniciar un nuevo proceso de registro.
Actor Solicitante / Usuario del Kiosco
Precondiciones El ciudadano debe haber solicitado un turno previamente en la fecha actual dentro del sistema DigiTurno.
Flujo Normal 1. El ciudadano selecciona "CONSULTAR TURNO" en el footer del kiosco.
2. El sistema abre un modal y solicita el número de documento de identidad.
3. El ciudadano ingresa su identificación y presiona "Buscar".
4. El sistema valida que el documento tenga un turno activo para el día de hoy.
5. El sistema consulta la BD y extrae: Código, Estado, Servicio, Perfil y Hora.
6. El sistema presenta los datos en una tarjeta informativa dentro del modal.
7. El ciudadano cierra la ventana tras visualizar su estado.
Postcondiciones El ciudadano queda informado sobre su turno sin generar un registro duplicado.
Flujos Alternativos A1: No existe turno → El sistema informa: "No se encontraron turnos activos para este documento hoy".
A2: Error de Red → El sistema notifica fallo de comunicación con el servidor.
Flujo Principal 1. Inicio de consulta. 2. Validación de identidad. 3. Recuperación de datos del turno vía GET /api/turno/consultar/{documento}. 4. Presentación de información al ciudadano en modal.

🔐 CU-06 — Gestión y Registro de Usuarios del Sistema
ID CU-06
Nombre Gestión y Registro de Usuarios del Sistema
Descripción Permite al Coordinador registrar nuevos usuarios (Asesores) en el sistema, definiendo su rol, credenciales de acceso y validando la integridad de los datos.
Actor Coordinador
Precondiciones El sistema debe estar activo y el Coordinador debe tener acceso al panel de control o a la URL de registro.
Flujo Normal 1. El Coordinador accede a "Gestión de Módulos" → "Nuevo Asesor".
2. El sistema despliega el formulario con campos: Tipo Doc, Documento, Nombres, Apellidos, Teléfono, Correo, Contraseña, Tipo de Asesor.
3. El Coordinador selecciona el Tipo de Asesor: OT (Orientador Técnico), OV (Orientador de Víctimas) o AT (Asesor Total).
4. El sistema valida formato de correo, longitud de contraseña (mínimo 6 caracteres) y unicidad del documento y correo.
5. El sistema crea la Persona y el Asesor en la BD con contraseña hasheada (bcrypt).
6. El sistema muestra mensaje de confirmación: "Asesor registrado exitosamente".
Postcondiciones El asesor queda creado en la BD con el tipo asignado (OT/OV/AT). El sistema queda disponible para que el nuevo asesor inicie sesión con sus credenciales.
Flujos Alternativos 1.a: Correo ya registrado → "Este correo electrónico ya está en uso por otro asesor".
1.b: Documento ya registrado → "Este número de documento ya está registrado como asesor".
1.c: Tipo de asesor no seleccionado → "Debe seleccionar el tipo de asesor".
1.d: Contraseña menor a 6 caracteres → Validación de Laravel bloquea el envío.
1.e: Auto-registro desde /asesor/register → El asesor puede registrarse directamente, inicia sesión automáticamente y redirige al panel.
Flujo Principal El Coordinador accede al módulo de registro. El sistema valida privilegios. Se capturan los datos del nuevo asesor. El sistema ejecuta la persistencia en BD (Persona + Asesor) con transacción DB::beginTransaction(). El sistema confirma el registro exitoso.
Tipos de Asesor OT — Orientador Técnico: Atiende General y Prioritario (relación 3:1)
OV — Orientador de Víctimas: Atiende Víctima y Empresario
AT — Asesor Total: Atiende los 4 perfiles con prioridad completa (Víctima → Empresario → Prioritario → General)
